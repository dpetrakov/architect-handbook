# ADR-0002: PostgreSQL как основная СУБД

**Дата:** 2025-03-15
**Статус:** accepted

## Контекст

Проекту нужна реляционная СУБД для хранения данных о заказах и клиентах. Ожидаемый объём — до 500 ГБ, нагрузка — до 1000 RPS на чтение, до 100 RPS на запись. Команда имеет опыт с PostgreSQL и MySQL.

## Рассмотренные варианты

### Вариант A: PostgreSQL

- Плюсы: опыт команды, JSONB для полуструктурированных данных, расширяемость, managed-версии во всех облаках РФ
- Минусы: сложнее горизонтальное масштабирование без Citus/шардирования

### Вариант B: MySQL

- Плюсы: проще в эксплуатации, широкая поддержка хостингов
- Минусы: слабее работа с JSON, меньше возможностей для аналитических запросов, у команды меньше опыта

## Решение

Выбран PostgreSQL. Опыт команды снижает риски. JSONB покрывает потребность в полуструктурированных данных без отдельного document store. Managed PostgreSQL доступен в Yandex Cloud и VK Cloud.

## Последствия

- ORM: используем без привязки к PostgreSQL-специфичным фичам (на случай миграции)
- Мониторинг: настраиваем pg_stat_statements и алерты на медленные запросы
- Бэкапы: ежедневные, WAL-архивирование для point-in-time recovery
- Принимаем риск: при росте нагрузки выше 5000 RPS потребуется шардирование или read-реплики
